Re-Ordering Columns

Layouts may need to be re-organized sometimes for smaller screens, but the source code is always written in one specific order. To change the order in which columns appear, simply add the .x-push.y and .x-pull-y classes to the columns in a row where you need to change the layout. Note (2nd and 3rd rows) that when applying .push / .pull classes to columns that also haven .offset class, the 'offset' amount remains present for all elements in the entire row. However, if you remove the .offset class on the first column in source order, the whole row shifts back to the left.

Structure & Implementation

The diagram and code example below show the necessary elements and hierarchy that is needed in order to create layouts using the grid.
This is the HTML element in your structure that will be the parent section of any content that you wish to align to the grid—i.e. Headers, Sections, Divs etc...
The very next element should be a div with the class of .content-wrap. This will be the master container of the grid, and will contain all grid elements and content.
The .content-wrap element can contain mulitple 'rows', which as its name implies, organizes your layout design into rows. These rows are necessary to control the layout of the children columns, which make up the actual grid.
Finally, the main element of the grid—columns. Syntax for the column classes is formatted as .x-col-y, where 'x'=[sm, md, or lg] and 'y'=[number of columns the element should span].
