Spyd r

A customizable, fluid CSS grid system for creating responsive layouts

Simple Structure

Implementing the grid is very straightforward and requires just three components.

1) .grid – Serves as the parent container for the entire grid.

2) .grid__row – Helps set the vertical rhythym of layouts. This layer should always be placed within the .grid element.

3) .grid__col – Use columns to build and control the horizontal flow of layouts. Columns should always be placed within the .grid__row element, and all of the content you wish to align to the grid should be placed within columns.

NOTE: There is an advanced syntax for columns, further explained in the Columns & Syntax section below.

<div class="grid">
<div class="grid__row">
<div class="grid__col">
...Your content goes here
</div>
</div>
</div>

Fluid

Spyder is built using flexbox and percentage widths. This allows for fluid resizing and responsiveness of the grid and columns.

.sm-col {
max-width: 100%;
flex-basis: 100%;
}
.md-col--4_8 {
max-width: 50%;
flex-basis: 50%;
}
.lg-col--3_12 {
max-width: 25%;
flex-basis: 25%;
}

Columns & Syntax

As mentioned above, there are two different implementations of columns, each with their own syntax:

  • 1) Auto Columns
  • 2) Responsive Columns

Auto Columns .grid__col

Auto Columns are the most basic form of columns, with their syntax following the BEM namespacing that Spyder uses by default.

NOTE: Auto Columns are fluid in width, but the only responsive behavior they exhibit is the ability to morph into vertically stacked, full-width columns at small device sizes. Because of this, it's recommended to use Auto Columns sparingly and/or for very basic layouts.

You can explore the behavior of Auto Columns by adding/removing columns below.

.grid__col
.grid__col
<div class="grid__row">
<div class="grid__col"></div>
<div class="grid__col"></div>
</div>

Responsive Columns .[x]-col--[y_z]
