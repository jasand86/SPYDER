Simple Structure

Implementing the grid is very straightforward and requires just three layers.

1) .grid – Serves as the parent container for the entire grid.

2) .grid__row – Helps set the vertical rhythym of layouts. This layer should always be placed within the .grid element.

3) .grid__col or .col--[x]-[y_z] – Use columns to build and control the horizontal flow of layouts. Columns should always be placed within the .grid__row element. ** It's important to note there are two syntax variations for columns—both explained in the 'Column Syntax' section below.

<div class="grid">
<div class="grid__row">
<div class="grid__col">
...Your site content goes here
</div>
</div>
</div>

Fluid

Spyder uses percent based widths, which allow for fluid resizing of columns and rows.

.col--sm {
max-width: 100%;
flex-basis: 100%;
}
.col--md-4_8 {
max-width: 50%;
flex-basis: 50%;
}
.col--lg-3_12 {
max-width: 25%;
flex-basis: 25%;
}

Columns & Syntax

As mentioned above, there are two(2) different variations of columns, each with their own syntax:

  • ¬ Auto Columns
  • ¬ Responsive Columns

Auto Columns .grid__col

Auto Columns are the most basic implementation of columns, and follow the BEM naming convention that Spyder uses by default. They are fluid in width, but are only responsive at small device sizes.

.grid__col
